Quick Start

Get AnchorCast running in under 5 minutes.

First-Time Setup

1
Upload the Bible database
Go to Settings → Bible Versions. Download kjv.json from the link shown, then click Upload next to KJV. This loads all 31,102 verses. You can also run data/download-bible.bat (Windows) or ./data/download-bible.sh (Mac/Linux).
2
Add your Claude API key (for AI detection)
Go to Settings → Audio & AI → General and paste your Claude API key. Get one free at console.anthropic.com. Without a key, AnchorCast uses keyword matching instead of AI.
3
Open the projection window
Click Projection in the top bar, or press Ctrl+P. Drag the projection window to your second screen or projector and make it fullscreen.
4
Start the transcript
Click Start Transcript at the bottom of the Live Transcript panel. Allow microphone access when prompted. The app will begin transcribing and detecting verses automatically.
5
Go Live and present verses
Click Go Live in the top-right corner. When a verse appears in AI Detections, click Present. It instantly appears on the projection screen.
💡
Manual mode tip: In Manual mode, clicking a verse in Bible Search shows it in Program Preview first. Double-click (or press Enter) to push it live. This lets you verify the verse before the congregation sees it.

Bible Setup

AnchorCast comes with 223 built-in key verses but requires a full Bible JSON file for complete access to all 31,102 verses.

Installing a Translation

  1. Open Settings → Bible Versions
  2. Click Upload next to any translation (e.g. KJV)
  3. Select the JSON file from your computer
  4. Restart AnchorCast — the translation loads automatically

Downloading Free Bible Data

Run the one-click script in the anchorcast/data/ folder:

  • Windows: Double-click download-bible.bat
  • Mac/Linux: Run ./download-bible.sh in Terminal

Or download manually from github.com/scrollmapper/bible_databases

Supported Translations

AnchorCast accepts any translation — not just the ones listed. To upload a custom translation:

  1. Name your file after the translation abbreviation: asv.json, gnt.json, etc.
  2. Or just drag any .json file onto the upload zone — AnchorCast will ask you to type the abbreviation

JSON Format

Bible JSON files must be an array of verse objects:

[
  {"b": 1, "c": 1, "v": 1, "t": "In the beginning God created..."},
  {"b": 1, "c": 1, "v": 2, "t": "And the earth was without form..."}
]

Where: b = book (1–66) · c = chapter · v = verse · t = text

Live Transcript

The Live Transcript panel shows real-time speech-to-text from your microphone. As the preacher speaks, AnchorCast transcribes every word and passes it to the AI for verse detection.

Starting Transcription

Click Start Transcript. If a microphone is not found, AnchorCast enters Demo Mode — it plays a pre-recorded sample sermon so you can see how the app works.

Audio Sources

Toggle between three source modes in the transcript panel header:

  • LIVE — Browser microphone (Web Speech API)
  • LOCAL — Local Whisper AI transcription (desktop only, requires Python)
  • CLOUD — Cloud-based transcription service

Audio Setup Tips

  • For best results, connect a USB audio interface to the mixing board's aux/monitor output
  • Select your audio device in Settings → Audio & AI → General
  • The mic level bars in the panel header animate when audio is being received

Online vs Offline Mode

Toggle Online / Offline in the titlebar:

  • Online: Uses Claude AI for best verse detection accuracy (requires API key + internet)
  • Offline: Uses fast local keyword matching — no internet needed, works anywhere

Saving Transcripts

When you stop transcription, the session is automatically saved to History. You can browse, export, and search past sermons there.

💡
Export: Click the export button in the transcript footer to save the current transcript as a text file.
💡
Sermon Notes: Once transcription is running, use Sermon Notes (Ctrl+Shift+N) to generate AI-powered structured notes from the live transcript.

AI Verse Detection

As the preacher speaks, AnchorCast's AI engine identifies Bible verses being referenced — including paraphrases, partial quotes, and indirect references.

How It Works

  1. Speech is transcribed in real time
  2. The transcript is sent to Claude AI (or local keyword matching in offline mode)
  3. Detected verses appear in the AI Detections panel with a confidence score
  4. Click Present to show the verse, or + Queue to stage it for later

Detection Methods

Online mode (API key + internet)

  • Direct regex — Exact citations like "John 3:16" (always active)
  • Learned phrases — User-curated phrase-to-verse mappings (always active)
  • Claude AI — Understands paraphrases, quotes, and contextual references

Offline mode (no API key or no internet)

  • Direct regex + Learned phrases (same as online)
  • Verbal cues — Recognises spoken patterns like "turn to", "the bible says in", "as David said"
  • Keyword signatures — ~80 regex patterns for famous verse phrases
  • Content search — Fuzzy n-gram matching against the Bible database

Confidence Scores

  • 85%+ — High confidence (green dot) — direct quote or very clear reference
  • 65–84% — Medium confidence (yellow dot) — likely reference
  • Below 65% — Low confidence (red dot) — possible reference

Present from Detections

When you click Present on a detected verse:

  • The verse is shown in Program Preview
  • If Go Live is ON, it also appears on the projection screen immediately
  • The Bible Search panel automatically navigates to that book and chapter so you can see the surrounding context

Auto Mode

Switch to Auto mode (click the Manual badge) to automatically present high-confidence verses (85%+) without any clicks.

💡
Confidence threshold: Adjust in Settings → Audio & AI → General → Confidence Threshold. Higher = fewer results but more accurate. Lower = more results but some may be wrong.

Bible Search

The Bible Search panel lets you manually find and present any verse from any installed translation.

Book Search

Type in the search box:

  • Book name only — e.g. John → shows John chapter 1
  • Book + chapter — e.g. John 3 → shows all verses in John 3
  • Full reference — e.g. John 3:16 → shows the chapter with verse 16 highlighted
  • Abbreviations — e.g. Rom 8, Ps 23, Isa 40:31

Context Search

Click the Context Search tab and type a topic or phrase (e.g. "strength", "God's love", "salvation"). If an API key is set, Claude finds the most relevant verses. Otherwise, local keyword matching is used.

Presenting Verses

  • Single click a verse row → shows it in Program Preview
  • Double click → sends directly to Live Display
  • Present button → Preview only
  • + button → Adds to Queue

Keyboard Navigation

  • ↓ or → — Next verse in search results
  • ↑ or ← — Previous verse
  • Enter — Push current preview verse to Live
  • Tab — Toggle between Book Search / Context Search

Chapter Navigation

Use the ‹ › buttons to move to the previous or next chapter. The chapter/book cycles through the entire Bible.

Display & Projection

AnchorCast drives two display areas: Program Preview (what you see) and Live Display (what the congregation sees).

Program Preview

Shows what verse, song, or media is staged and ready. The congregation does NOT see this. Use it to check content before pushing it live. Switch between List view and Slides view using the toggle.

Live Display

The content currently showing to the congregation. When Go Live is ON:

  • Clicking any verse in Bible Search immediately pushes it to Live Display
  • The projection screen updates instantly

Go Live Toggle

  • OFF (Go Live) — You can stage content in Preview without affecting the projection screen
  • ON (On Air) — Every selection goes straight to the projection screen. A red ON AIR tag appears on the Live Display.
  • Pressing Go Live → OFF blanks the projection screen immediately

Projection Window

Click Projection or press Ctrl+P to open the projection window. Drag it to your second screen and press F11 (Windows/Linux) to make it fullscreen.

Content Types

The projection screen can display:

  • Scripture — Bible verses with translation reference
  • Songs — Lyrics slides with slide-by-slide navigation
  • Media — Images and video backgrounds
  • Presentations — Custom slides from the Presentation Editor
💡
Translation change: Changing the translation dropdown while ON AIR immediately re-renders the current verse in the new translation on the projection screen.

Overlaying Songs & Scripture on Media

AnchorCast can overlay song lyrics or Bible verses on top of a background video or image — perfect for worship backgrounds with lyrics, or displaying scripture over scenic footage.

How It Works

When a background media (video or image) is playing on the projection screen, you can layer song lyrics or scripture text on top of it. The background dims slightly so the text remains readable, and the congregation sees both the visual media and the text together.

Setting Up a Media Overlay

1
Add a background video or image
Go to the Media tab in the bottom panel. Click a video or image to load it into Program Preview, then present it live. This becomes your background.
2
Enable text overlay
Click the 🖼+T button in the Program Preview toolbar. When active, it shows 🖼+T On. This is the master toggle for all text overlays on media.
3
Choose what to overlay
Use the individual toggle buttons next to the master toggle:
• 🎵+ — Enable song lyrics overlay on media
• ✝+ — Enable scripture overlay on media
You can enable one or both at the same time.
4
Present your song or verse
Now select a song from the Songs tab or a Bible verse from Bible search and present it. The text appears overlaid on the media background instead of a plain background.

Toolbar Quick-Toggle Buttons

The Program Preview toolbar provides four overlay buttons for quick access during a live service:

  • 🖼+T (Master toggle) — Turns text-on-media overlay on or off. When off, songs and verses display on their own themed background as usual.
  • 🎵+ (Song toggle) — Controls whether song lyrics appear over the background media. Must be enabled along with the master toggle.
  • ✝+ (Scripture toggle) — Controls whether Bible verses appear over the background media. Must be enabled along with the master toggle.
  • ▁T (Lower-third mode) — When enabled, text appears as a lower-third bar at the bottom of the screen instead of centered. Great for keeping the background media visible while showing smaller text at the bottom.

Settings Panel (Settings → Main Output)

For persistent configuration, open Settings → Main Output and scroll to the Overlay on Media section. This is where you configure overlay behaviour that persists across sessions:

  • Overlay songs / scripture on projected media — Master switch. When enabled, text is kept on the current video or image in the background when sending scripture or songs live.
  • Allow song overlay — Songs can remain over active image or video backgrounds.
  • Allow scripture overlay — Scripture can remain over active image or video backgrounds.
  • Use lower-third mode — Place overlay text lower on the screen instead of centered.

Background Dim Behind Overlaid Text

Below the overlay checkboxes in Settings → Main Output, a Background Dim slider controls the darkness added over the active background for readability. The default is 35%. Drag it higher to darken the background more for better text contrast, or lower to keep the background brighter. This setting is saved automatically.

Example: Worship Song Over a Video

1
Present a worship background video
From the Media tab, click a looping worship video (e.g. clouds, water, abstract visuals) and present it live.
2
Turn on overlay
Click 🖼+T to enable overlay, then click 🎵+ to enable song overlays.
3
Present a song
Switch to the Songs tab, select your worship song, and present it. The lyrics appear on top of the video background.
4
Navigate slides
Use → / ← arrow keys to move through song slides. Each slide overlays on the same background video.
💡
Mix and match: You can overlay scripture on media during the sermon (enable ✝+), then switch to song overlays (enable 🎵+) during worship — all on the same background video. The toggles are independent so you can switch without interrupting the media.
💡
Lower-third for sermons: During scripture reading, enable ▁T lower-third mode so the verse appears in a bar at the bottom of the screen, keeping the main visual area unobstructed. This works well when showing a speaker camera feed or scenic background.

Queue & Schedule

The Schedule panel (middle column) lets you build a service order and stage multiple items in advance — verses, songs, media, and presentations.

Adding to the Schedule

There are several ways to add items to the schedule:

  • Click the + button on any verse in Bible Search or AI Detections
  • Click +📋 on any song in the Songs tab to add it to the schedule
  • Click + on any media item in the Media tab
  • Click + Schedule on any presentation in the Presentations tab

Drag & Drop

You can drag content directly from the content panels into the schedule:

  • Bible verses — Drag any verse row from the Bible Search results and drop it onto the schedule
  • Songs — Drag a song from the Songs tab into the schedule
  • Media — Drag images, videos, or audio from the Media tab into the schedule

When you drop an item onto an existing schedule item or section header, the new item is inserted directly below the drop target. If you drop onto the empty area at the bottom of the schedule, the item is appended to the end.

A gold highlight indicator shows where the item will be placed as you drag over the schedule.

Reordering Items

Items already in the schedule can be rearranged by dragging:

  • Grab the dotted drag handle on the left edge of any schedule item
  • Drag it up or down to a new position
  • Release to drop it in place — the currently active item index is tracked automatically

Section Headers

Organise your service order with section dividers:

  • Click the § button in the Schedule panel header to add a section divider
  • Type a label (e.g. "Worship", "Sermon", "Announcements") directly into the section header
  • Section headers are visual dividers — they are skipped when navigating with arrow keys
  • You can drag and drop content onto a section header to insert it below that section

Deleting Items

  • Click the ✕ button on any schedule item to remove it
  • Songs and media can also be deleted from their library tabs using the 🗑 button — a confirmation dialog will appear before deletion

Schedule Item Types

The schedule supports a mix of content types:

  • Scripture — Bible verses with translation and reference
  • Songs — Full songs with slide-by-slide navigation
  • Media — Images and videos for worship backgrounds or announcements
  • Presentations — Custom slide decks from the Presentation Editor

Auto-Play

The Auto button in the Schedule panel header lets you auto-play through all schedule items hands-free:

  • Click ▶ Auto to start — the button turns red and shows ⏹ Stop
  • Each item type has a different hold time:
    • Images — 5 seconds
    • Bible verses — 8 seconds
    • Songs — 10 seconds per slide
    • Videos / Audio — Waits for the media to finish playing
  • Click ⏹ Stop at any time to halt auto-play
  • A toast notification appears when auto-play finishes the last item

Presenting from Schedule

  • Click any schedule item to present it in Program Preview
  • Press → / ↓ to go to the next item
  • Press ← / ↑ to go to the previous item
  • Press Enter to push the current preview to Live

Saving & Loading Schedules

Schedules can be saved for reuse — ideal for recurring services or pre-planned worship sets:

  • Save — Ctrl+S or File → Save Schedule
  • Save As — Ctrl+Shift+S or File → Save Schedule As
  • Open — Ctrl+O or File → Open Schedule
  • New — Ctrl+N or File → New Schedule to start a fresh service order

Schedule files are saved as .json files and can be shared between AnchorCast installations.

💡
Keyboard flow: Arrow keys first navigate within Bible Search results. If no search results are active, they navigate the Schedule instead — allowing completely hands-free operation.
💡
Auto Service Builder: Use Auto Service Builder to automatically generate a schedule based on your church's service history and patterns.

Song Manager

Manage your church's full song library — create, edit, import, and search songs with automatic slide formatting.

Opening the Song Manager

Open from Tools → Song Manager (Ctrl+M) or click the Songs button in the titlebar.

Creating a Song

  1. Click + New Song
  2. Enter the song title, artist, and key
  3. Type or paste lyrics in the editor
  4. Use blank lines between sections to create separate slides (2 lines = 1 slide)
  5. Click Save

Slide Formatting

AnchorCast automatically splits lyrics into slides. Each pair of lines becomes one slide. To control slide breaks:

  • Leave a blank line between sections (Verse 1, Chorus, etc.)
  • Add section labels like [Verse 1], [Chorus], [Bridge]
  • The rich text editor supports bold, italics, and lists for individual slides

Genius Lyrics Search

Click Search Lyrics in the Song Manager header to open the lyrics search overlay:

  1. Type a song name or artist
  2. Results appear from Genius with song title, artist, and album art
  3. Click a result to preview the lyrics
  4. Click Add to Library to import with auto-parsed slides
💡
Genius API key: Lyrics search requires a Genius API key. Get one free at genius.com/api-clients and add it in your environment variables or settings.

Importing & Exporting

  • Import JSON — Import songs from AnchorCast JSON format
  • Import TXT — Import plain text lyrics files
  • Export JSON — Export your entire song library

Importing from Other Worship Software

AnchorCast can import songs from EasyWorship, ProPresenter (all versions including 7), plain text files, and SQLite databases. Go to Settings → Songs → General → Song Library Management → Import from other app.

For full step-by-step instructions, supported file types, and tips, see Settings → Importing Songs from Other Worship Software.

Presenting Songs

From the main dashboard, use the Songs tab in the bottom panel to browse your library. Click a song to load it into the Preview, then use Go Live or the Present button. Navigate slides with arrow keys.

Presentation Editor

Create custom announcement slides, worship backgrounds, and visual content with the built-in canvas editor.

Opening the Editor

Open from Tools → Presentation Editor (Ctrl+E) or from the Presentations tab in the bottom panel.

Canvas Tools

The editor provides a 1920x1080 canvas with these tools:

  • Text — Add titles and body text with font, size, and colour controls
  • Images — Insert photos, logos, and graphics
  • Shapes — Banners, speech bubbles, crosses, and basic shapes
  • Background — Set a solid colour or image background for each slide

Managing Slides

  • The sidebar shows all slides in order
  • Drag slides to reorder them
  • Click + to add a new slide
  • Right-click a slide for duplicate/delete options

Saving & Presenting

Click Save to save the presentation. It appears in the Presentations tab on the main dashboard. Click it to load into Preview, then present with Go Live.

Import & Export

  • Export — Save as AnchorCast presentation JSON
  • Import — Load from JSON file
  • PPTX Import — Import PowerPoint files (desktop only, requires Python)

Theme Designer

Customise how content looks on the projection screen with built-in themes or your own designs.

Theme Categories

AnchorCast supports separate themes for different content types:

  • Scripture — How Bible verses appear on screen
  • Songs — How song lyrics appear
  • Presentations — Backgrounds for presentation slides

Built-in Themes

  • Sanctuary — Deep purple/gold gradient, classic church look
  • Dawn — Warm brown tones
  • Deep Water — Cool blue tones
  • Minimal — Pure black background, white text

Creating a Custom Theme

  1. Open Themes from the titlebar (Ctrl+T)
  2. Click + New — a copy of the current theme is created
  3. Type a name in the rename field that appears in the sidebar
  4. Adjust Text, Background, and Layout settings in the properties panel
  5. Click Save to save your theme

Theme Properties

  • Font — Choose from Cinzel, DM Sans, Crimson Pro, and more
  • Text size, alignment, and shadow
  • Background colour — Solid or gradient
  • Background image — Upload a custom image
  • Background video — Use a looping video behind text
  • Overlay — Add a colour overlay on top of background media

Quick Theme Switching

Use the theme swatches in the titlebar to instantly switch between saved themes during a live service — no need to open the Theme Designer.

Deleting a Theme

Select a custom theme in the sidebar. A delete button appears next to the rename field. Click it and confirm to delete. Built-in themes cannot be deleted.

Background

Set a persistent background image or video that appears behind all projected content — scripture, songs, and presentations.

Setting a Background

1
Open the Live Display panel
The Live Display panel is on the right side of the main window. Look for the BG button in the panel header.
2
Click BG
Click the BG button to open a file picker. Select an image or video from your media library to use as the persistent background.
3
Background appears on projection
Once set, the background appears behind all content on the projection screen. Scripture and song text will display on top of this background.

Clearing the Background

Click the BG button again and choose to clear/remove the background, or select a different image or video to replace it.

Background vs Media

There is an important distinction between a background and presenting media:

  • Background — Persists behind all content. Scripture and songs overlay on top of it. It stays until you explicitly clear it.
  • Presenting media — A one-time display of an image or video. When you present a different item, the media changes.
💡
Save with Presets: Your background setting is saved as part of Presets, so you can quickly switch between different service looks including their backgrounds.

Logo Overlay

Add your church's logo as a persistent overlay on the projection screen — it appears on top of all content including backgrounds, scripture, songs, and media.

Adding a Logo

1
Click LOGO in the Live Display panel
Look for the LOGO button in the Live Display panel header (top-right area). Click it to open the logo settings.
2
Upload your logo image
Click Choose Logo to select a PNG or image file from your computer. Transparent PNG files work best — the transparent areas will show the content behind them.
3
Position your logo
Choose from preset positions: Top-Left, Top-Right, Bottom-Left, Bottom-Right, Center, or Drag for custom placement.

Logo Controls

  • Position — Choose a corner, center, or drag freely to any position on the projection screen
  • Size — Adjust the logo size using the size slider
  • Opacity — Control the logo's transparency with the opacity slider — useful for a subtle watermark effect
  • Show / Hide — Toggle the logo visibility on or off without removing it

Custom Drag Positioning

When the position is set to Drag, you can freely move and resize the logo on the projection window:

  • Click and drag the logo to reposition it anywhere on the screen
  • Use the corner handle to resize the logo
  • The position and size are saved and sync back to the controller
💡
Save with Presets: Logo settings (image, position, size, opacity, visibility) are all saved as part of Presets. Switch between different logo configurations for different services or events.
💡
Best format: Use a transparent PNG for your logo so it blends seamlessly over any background or content.

Presets

Save and load named configurations of your entire service look — background, logo, themes, schedule, and settings — so you can switch between different setups instantly.

What a Preset Saves

Each preset captures:

  • Background — The current background image or video
  • Logo Overlay — Logo image, position, size, opacity, and visibility
  • Themes — Scripture theme, song theme, and presentation theme selections
  • Schedule — The entire service order including all items and the schedule name
  • Settings — All application preferences (API keys are excluded for security)

Saving a Preset

1
Set up your service look
Configure your background, logo, themes, and schedule the way you want them.
2
Click the star button
Click the ☆ button in the Live Display panel header. A preset popup appears.
3
Click Save Current
Click Save Current in the preset popup. Enter a name for your preset (e.g. "Sunday Morning", "Youth Night", "Christmas Service").
4
Confirm the save
Click Save to store the preset. It will now appear in your preset list.

Loading a Preset

  • Click the ☆ button to open the preset popup
  • Click any preset name to load it immediately
  • All saved settings — background, logo, themes, schedule, and preferences — are restored

Managing Presets

Open File → Manage Presets or click Manage in the preset popup to open the Manage Presets window:

  • Load — Click the load button to apply a preset
  • Delete — Click the delete button, then confirm to permanently remove the preset

Accessing Presets from the Menu

Presets are also available from the File menu:

  • File → Save Preset — Save the current configuration as a new preset
  • File → Load Preset — Choose from your saved presets to load
  • File → Manage Presets — Open the full preset manager
💡
Ideal for recurring services: Save a preset for each type of service your church runs — Sunday morning, evening service, youth night, holiday events. Before each service, load the appropriate preset and everything is configured instantly.
💡
Security note: API keys are never stored in presets. When you load a preset, your current API key and sensitive credentials are preserved.

Remote Control

Control AnchorCast from any phone or tablet on the same WiFi network — no app installation required.

Setup

  1. Make sure the remote server is ON (green) in the Remote button popover
  2. On your phone, open a browser and go to the URL shown — e.g. http://192.168.1.5:8080/remote
  3. The mobile remote page loads in the browser — no app install needed

Remote Features

  • Live projection screen preview on the phone
  • Mode tabs for Scripture, Songs, and Media
  • Search and quick-verse lookup
  • Song and media library browsing
  • Prev/Next/Clear/Go Live controls

Remote Actions

  • Go Live / Off Air — Toggle the projection on/off
  • Next / Previous — Navigate the queue
  • Clear — Blank the projection screen
  • Present — Send selected content to projection

Authentication (Desktop)

In Electron desktop mode, the remote control uses PIN-based authentication with role-based access:

  • Admin — Full control over all features
  • Scripture — Can present Bible verses only
  • Songs — Can present songs only
  • Media — Can present media only
  • Monitor — View-only access (can see the projection but not control it)

Enabling / Disabling

Click Remote in the titlebar to open the remote popover. Use the toggle switch to enable or disable the server. You can also toggle it in Settings → Network.

⚠️
The remote control server is accessible to anyone on your WiFi network. Disable it when not in use if you're on a public or shared network.

Wrong IP Address?

If the IP shown is incorrect (e.g. a VPN address), check that your device is connected to the correct WiFi network. AnchorCast automatically picks the best available network adapter, preferring real Ethernet/WiFi over VPN or virtual adapters.

NDI & External Output

Send AnchorCast's projection output directly to OBS, vMix, Wirecast, or any NDI-capable software on the same network. Desktop only.

How NDI Works

When NDI is active, AnchorCast appears as an NDI source called "AnchorCast" on your network. In OBS, simply add an NDI Source and select AnchorCast. No URL configuration needed — it's automatic network discovery.

Setting Up NDI

1
Install the NDI SDK
Windows: NDI 6 SDK from ndi.video → installs to C:\Program Files\NDI\NDI 6 SDK. macOS: NDI SDK for Apple → installs to /Library/NDI SDK for Apple/.
2
Build the NDI addon
Windows: Run ndi-addon\build-ndi.bat (requires Visual Studio Build Tools). macOS: Run bash ndi-addon/build-ndi.sh (requires Xcode Command Line Tools).
3
Start NDI output
In AnchorCast, go to the External Output panel (from the menu or popover). Click Start. When NDI is ready, the status shows "NDI active".

Using in OBS

  • Add Source → NDI Source → select AnchorCast
  • The feed shows whatever is on the projection screen in real time
  • Works across machines on the same network

Using in vMix

  • Add Input → NDI / Desktop Capture → select AnchorCast

MJPEG Fallback

If the NDI addon is not built, AnchorCast falls back to an MJPEG HTTP stream:

  • OBS: Add Source → Browser Source → http://localhost:49876/stream
  • vMix: Add Input → Web Browser → http://[PC-IP]:49876/stream
💡
Automated build: When running npm run build:win or npm run build:mac, the pre-build script automatically attempts to compile the NDI addon. Set SKIP_NDI_BUILD=1 to skip.

Verse Detection Review

Review and improve verse detections to teach AnchorCast what your preacher's speaking patterns mean.

Review Actions

When a verse is detected, you can take these actions from the AI Detections panel:

  • Approve — Confirm the detection was correct. This teaches the system to be more confident about similar detections.
  • Reject — Mark it as a false positive. Helps the system avoid similar mistakes.
  • Replace — The detection was close but the wrong verse — select the correct one. Teaches the system the right mapping.
  • Teach — Save the source phrase so AnchorCast recognises it next time (adds to learned phrases).

Learned Trigger Phrases

When you teach a phrase, AnchorCast saves the mapping (e.g. "the Lord is my shepherd" → Psalm 23:1). Next time the preacher says it, the verse is detected instantly without needing AI — even in offline mode. This is one of the most powerful features for building accuracy over time.

Managing Learned Data

View and manage all learned phrases and review history in the Adaptive Memory management window. You can edit, delete, or export learned phrases from there.

Data Storage

Review events and learned phrases are stored as portable JSON files:

  • Data/detection_review/learned_phrases.json — All taught phrase-to-verse mappings
  • Data/detection_review/review_events.json — History of approve/reject/replace actions
💡
Review regularly: Spending a few minutes reviewing detections after each service significantly improves accuracy. The system learns fastest from consistent feedback.

Importing Songs from Other Worship Software

AnchorCast can import your existing song library from other popular worship presentation software. Go to Settings → Songs → General → Song Library Management and click Import from other app.

Supported Sources

  • EasyWorship — Select the EasyWorship DATA folder (contains Songs.DB and Songs.MB). Use "Choose Folder" and navigate to the EasyWorship data directory (typically C:\Users\Public\Documents\Softouch\EasyWorship\Default\Databases\Data).
  • ProPresenter — Select individual .pro, .pro4, .pro5, .pro6, or .proPresenter song files. Use "Choose File" and select one or more ProPresenter song files.
  • Plain Text / RTF — Any .txt or .rtf lyrics files with one song per file.
  • SQLite Databases — Song databases from other applications (.db, .sqlite, .sqlite3).

How to Import

  1. Open Settings (Ctrl+,)
  2. Go to Songs in the sidebar
  3. Under the General tab, scroll to Song Library Management
  4. Click Import from other app
  5. Choose "Choose File" for individual song files (ProPresenter, TXT, JSON) or "Choose Folder" for an entire library folder (EasyWorship DATA folder)
  6. Select your files or folder — songs are imported and merged into your library

Import Modes

  • Merge — Adds new songs alongside your existing library. Duplicates are detected and skipped.
  • Replace — Replaces your entire song library with the imported songs. Use with caution.
💡
ProPresenter tip: Song files are typically in ~/Documents/ProPresenter/Libraries/ (macOS) or C:\Users\[username]\Documents\ProPresenter\Libraries\ (Windows). You can select multiple files at once.
💡
EasyWorship tip: Select the DATA folder itself, not individual files. The importer needs both Songs.DB and Songs.MB to extract full lyrics.

Troubleshooting

  • No Bible text on screen — check Bible Versions and confirm KJV or your chosen translation is loaded.
  • Nothing appears on projection — confirm content is previewed first and Go Live is enabled.
  • Song import looks wrong — use the dedicated import flow for song files.
  • Remote control fails — confirm phone and PC are on the same network.
  • NDI not appearing in OBS — check that the NDI addon is built and the NDI Runtime is installed.

Verses show placeholder text instead of Bible text

The full Bible database has not been loaded. Go to Settings → Bible Versions and upload a kjv.json file. See Bible Setup for instructions.

Transcription not working / no microphone

  • Check that your microphone is connected and selected in Settings → Audio & AI → General
  • Allow microphone permissions when the browser security prompt appears
  • If no microphone is found, AnchorCast runs in Demo Mode with a simulated sermon
  • On Windows, check that microphone privacy settings allow desktop apps

AI detection not working

  • Check your Claude API key in Settings → Audio & AI → General
  • Make sure Online mode is selected in the titlebar
  • If the API key is valid but detection is slow, switch to Offline mode for faster local keyword matching

Projection window shows blank screen after Go Live

Make sure a verse is loaded in Program Preview before turning Go Live ON. The projection only clears when Go Live is turned OFF — this is intentional to blank the screen cleanly.

NDI not working

  • Make sure the NDI 6 SDK is installed (Windows: C:\Program Files\NDI\NDI 6 SDK, macOS: /Library/NDI SDK for Apple/)
  • Build the addon: run build-ndi.bat (Windows) or bash build-ndi.sh (macOS) in the ndi-addon folder
  • Check that the NDI Runtime is installed on the receiving machine (OBS/vMix PC)
  • Both machines must be on the same network
  • If NDI fails, AnchorCast automatically falls back to MJPEG streaming

Genius lyrics search not finding results

  • A Genius API key is required — get one free at genius.com/api-clients
  • Make sure the key is set in your environment variables
  • Try searching with just the song title or just the artist name

Remote control shows wrong IP address

  • Ensure your PC is connected to the same WiFi network as the phone
  • If you have a VPN active, disable it — AnchorCast prefers real network adapters but a VPN can interfere
  • Check the IP shown in Remote popover matches your machine's actual LAN IP

Settings window won't open

Try pressing Ctrl+, or use Settings → Preferences from the top menu bar.

App freezes or crashes

  • Restart the app — transcript history is auto-saved
  • Check that node_modules folder exists. If not, run npm install in the AnchorCast folder
  • Make sure Electron version is compatible: run npx electron --version

Songs not displaying correctly

  • Check slide formatting — use blank lines between sections for slide breaks
  • Long lines may wrap differently on the projection screen depending on the theme's font size
  • Try switching to Slides view in Program Preview to check the layout before presenting
  • Check font and sizing settings in Settings → Songs → Font

Timer not showing on projection

  • Make sure Go Live is enabled before starting the timer
  • Check that the projection window is open (Ctrl+P)
  • Try changing the timer position (Centre, Top, or Edge) to confirm it's not hidden behind other content

Alerts not appearing on projection

  • Ensure Go Live is active and the projection window is open
  • Check alert settings in Settings → Alerts — the alert may be configured with a very short duration or small size
  • Try sending from the Tools menu (Tools → Alerts) instead of the inline control bar panel

Sermon notes generation fails

  • A Claude API key is required — check it's configured in Settings → Audio & AI → General
  • Make sure there is transcript text available — start a transcription session first
  • If the service was short, the AI may not have enough content to generate structured notes

ProPresenter song import not working

  • Ensure you're selecting the correct file types: .pro, .pro4, .pro5, .pro6, or .propresenter
  • ProPresenter 7 files use a different format — AnchorCast supports both XML-based and newer formats
  • Try importing one file at a time to identify problematic files
  • See Settings → Importing Songs from Other Worship Software for full details

Schedule not saving

  • Use Ctrl+S or File → Save Schedule to save
  • Check that the save location has write permissions
  • Try File → Save Schedule As to save to a different location

Adaptive learning not improving

  • Learning suggestions must be manually approved — open Settings → Audio & AI → Learning and review pending suggestions
  • More services means more data to learn from — the system improves over time
  • Check that adaptive data files exist in Data/adaptive/
